Actualizado start.sh con dos modos:
./start.sh          # modo desarrollo (tsx watch, hot-reload)
./start.sh dev      # modo desarrollo
./start.sh prod     # modo producción (build + <50MB RAM + logs JSON)
En modo prod ejecuta npm run build y luego npm run start:prod con los flags --max-old-space-size=50 --optimize-for-size --gc-interval=100 --expose-gc. En modo dev mantiene el comportamiento original con tsx watch.
