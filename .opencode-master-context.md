## 1. Arquitectura Desplegada

### 1.1 Multi-Tenant Isolation (WS-001)
- **Middleware**: `resolveTenant` hook extrae `X-Tenant-Slug` del header → `request.tenantSlug`
- **Columnas `tenant_slug`** añadidas a: `vehiculos`, `ordenes_trabajo`, `clients`, `facturas`
- **Aislamiento en queries**: Todo SELECT/UPDATE/DELETE filtra con `eq(tabla.tenantSlug, slug)`
- **Aislamiento en inserts**: Toda creación asigna `tenantSlug` desde el request

### 1.2 Motor de Facturación Híbrido (SIFEN-004-HYBRID)
```
POST /finance/invoices/issue
├── MANUAL:      numeroFacturaManual, sifen_cdc=NULL, sifen_status=MANUAL_CONVERT_QUEUE
└── ELECTRONICA: SifenCryptoService.signInvoiceAsync(), cdc extraído del XML firmado
```
- **Tabla `facturas`**: `tenantSlug`, `ordenId`, `tipo` (MANUAL|ELECTRONICA), `numeroFacturaManual`, `sifenCdc`, `sifenStatus`, `xmlRaw`, `xmlSigned`, `total`
- **Atómico**: transacción Drizzle única (crea factura, actualiza orden)
- **Transicional**: MANUAL queda en cola para conversión retroactiva cuando llegue homologación DNIT

### 1.3 Worker Criptográfico X.509 (SIFEN-002)
- **Worker thread** `src/finance/sifen-crypto.worker.ts`: firma SHA-256 + RSA en hilo aislado
- **Servicio `signXMLInWorker`** en `src/modules/finance/services/sifen/sifen-crypto.service.ts`: spawn, sign, terminate — cero hilos persistentes
- **ESM-safe**: `fileURLToPath(import.meta.url)` para resolución de ruta, `--import tsx/esm` para workers TypeScript

### 1.4 Canal de Ingesta Thinkcar (TC-002, TC-004, TC-005)
- **Email Worker** `thinkcar-email.service.ts`: IMAP vía `imapflow`, polling 5 min, backoff exponencial, lazy import
- **PDF Parser** `thinkcar-parser.service.ts`: `pdf-parse` + regex para extraer VIN, placa, DTCs, header, filename parsing (2 formatos), fallback regex
- **Smart Linker** `thinkcar-linker.service.ts`: VIN fuzzy matching, vinculación a vehículo/cliente/OT
- **Pipeline** `thinkcar-pipeline.service.ts`: pipeline orchestrator que conecta parser → linker → import
- **Notificaciones** `thinkcar-notifications.service.ts`: alerta email vía `nodemailer` cuando un reporte entra en `manual_review` (SMTP on-demand, `transporter.close()` en finally)

### 1.5 Cierre Contable Mensual (ACC-001)
- **`AccountingClosureService.executeMonthlyClosure(tenantSlug, year, month)`**
- Agregación vía `COALESCE(SUM(...))` en PostgreSQL — 0 filas en heap de Node
- Genera asiento contable (`asientosContables`) con doble entrada:
  - Débito → Caja/Banco (cuenta 1.1.01)
  - Crédito → Ingresos por Servicios (cuenta 4.1.01)
- Partidas en `asientosDetalle` para contabilidad completa
- `numero` auto-incremental por período via `COALESCE(MAX(...),0)+1`

## 4. Decisiones Arquitectónicas Clave

| Decisión | Razón |
|----------|-------|
| `facturas` separada de `fiscalDocumentos` | Tabla simplificada para período transicional híbrido; `fiscalDocumentos` es el esquema DTE completo para cuando la homologación DNIT esté activa |
| `signXMLInWorker` existente (no el SIFEN-002 simplificado) | El módulo finance ya tenía un servicio criptográfico completo con carga de certificados, worker inline y caché — se reutiliza |
| `db()` como función (no export directo) | Singleton lazy: la conexión postgres no se crea hasta el primer uso, manteniendo RAM <50MB al arranque |
| Parser tests sobre `parseFromPdf` mockeando `pdf-parse` | Evita necesidad de archivos PDF reales como fixtures |
| `numeroFacturaManual` en formato `001-001-0001234` | Sigue el estándar paraguayo de factura preimpresa: serie + número |
